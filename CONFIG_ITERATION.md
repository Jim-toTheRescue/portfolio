# 配置化

## 目标

功能不变，只是把参数变成可配置的。

## 数据结构

```javascript
{
  name: '第一梯队',  // 名称
  target: 30,       // 目标占比
  limit: 1,         // 主位上限
  buffer: 0,       // 缓冲位上限
  min: 25,         // 最小占比
  max: 35          // 最大占比
}
```

## 配置锁定

配置保存后锁定，无法再次修改。如需修改需手动清除 localStorage。

## 改动范围

| 文件 | 改动 |
|------|------|
| `constants.js` | 从 localStorage 读取，无则默认 |
| `helpers.js` | getTargetTier 动态判断 |
| `portfolio.js` | autoRebalance 循环用 TIER.length |
| UI组件 | 动态渲染 |

## 缓冲位使用规则

当目标梯队主位满了，缓冲位是否可用取决于更高梯队的主位空位数：

```
缓冲位可用 = 所有更高梯队的主位空位之和 > 0
```

例如：
- 2梯队配置：T1.limit=1, T2.limit=2
- T2 主位满了要进缓冲位：检查 T1 主位空位数
- 如果 T1 有空位（未满），T2 缓冲位可用

**缓冲位可用数量**：
- 可用数量 = 所有更高梯队的主位空位之和
- 当前梯队缓冲位数量不能超过可用数量

这样配置化后自动适配任何梯队数量。
