# 调仓按钮说明

> **重要**：以下说明基于配置化系统。梯队数量、目标比例、主位/缓冲位数量、min/max值均可配置。本文档以3梯队默认配置为例说明逻辑，实际行为根据配置动态生成。

## 配置数据结构

```javascript
{
  name: '第一梯队',  // 名称
  target: 30,       // 目标占比
  limit: 1,         // 主位上限
  buffer: 0,        // 缓冲位上限
  min: 25,          // 最小占比
  max: 35           // 最大占比
}
```

## 最重要限制

每一个按钮推荐的股票数，都必须最接近梯队规则限制值，也就是说如果推荐adjShares那么这个值交易出来的新仓位需要比
adjShares-1和adjShares+1更接近梯队限制值。而adjShares-1和
adjShares+1要么比adjShares离梯队限制值更远，要么直接溢出当前梯队了。

## 按钮规则（通用）

### 加仓面板按钮

| 梯队 | 按钮 | 目标值 |
|------|------|--------|
| T1 | = | target |
| T1 | max1 | max |
| T2 | ↑ | 上一梯队(T1) target |
| T2 | = | target |
| T2 | max1 | T1.max |
| T2 | max2 | max |
| T3+ | ↑ | 上一梯队 target |
| T3+ | = | target |
| T3+ | maxN | 当前梯队 max |

### 减仓面板按钮

| 梯队 | 按钮 | 目标值 |
|------|------|--------|
| T1 | = | target |
| T1 | min1 | min |
| T2 | = | target |
| T2 | min2 | min |
| T3 | = | target |
| T3 | min3 | min |
| T3 | ↓ | 下一梯队 target |
| T4+ | = | target |
| T4+ | minN | 当前梯队 min |
| T4+ | ↓ | 下一梯队 target |

---

## 按钮功能详解

### =（等于）按钮

调整到当前梯队目标比例 (target)。

- 当前持仓 > target → 减仓至 target
- 当前持仓 < target → 加仓至 target
- 结果：留在当前梯队

### ↑（向上箭头）按钮

调整到上一梯队（更高梯队）的目标比例。

- T1：不可用（无上一梯队）
- T2+：加仓至上一梯队 target，结果晋级

### ↓（向下箭头）按钮

调整到下一梯队（更低梯队）的目标比例。

- 最低梯队：不可用（无下一梯队）
- 其他梯队：减仓至下一梯队 target，结果降级

### maxN 按钮

加仓到当前梯队上限 (max)，结果留在当前梯队。

### minN 按钮

减仓到当前梯队下限 (min)，结果留在当前梯队。

---

## 按钮显示规则

根据当前持仓的梯队动态显示按钮：

- **T1 持仓**：=, max1（加仓）| =, min1（减仓）
- **T2 持仓**：↑, =, max1, max2（加仓）| =, min2（减仓）
- **T3 持仓**：↑, =, max2, max3（加仓）| =, min3, ↓（减仓）
- **T4+ 持仓**：↑, =, max{N-1}, maxN（加仓）| =, minN, ↓（减仓）

---

## 梯队槽位分配规则

### 槽位限制（根据配置）

- 每个梯队有 limit 个主位
- 每个梯队有 buffer 个缓冲位

### 升降级分配逻辑（通用规则）

1. 首先尝试进入目标梯队主位（如果主位有空位）
2. 如果主位已满，尝试进入目标梯队缓冲位（如果缓冲位有空位，且满足特殊检查）
3. 如果主位和缓冲位都满了，保持原位

### 缓冲位使用规则

当目标梯队主位满了，缓冲位是否可用取决于更高梯队的主位空位数：

```
缓冲位可用 = 所有更高梯队的主位空位之和 > 0
```

例如 2 梯队配置：T1.limit=1, T2.limit=2
- T2 主位满了要进缓冲位：检查 T1 主位空位数
- 如果 T1 有空位（未满），T2 缓冲位可用

---

## 边界规则

- 比例 ≥ 上一个梯队的 min 时，属于上一个梯队
- 例如：min=25 属于 T1，min=15 属于 T2，min=5 属于 T3

## 按钮汇总表（3梯队示例）

| 按钮 | T1持仓 | T2持仓 | T3持仓 |
|------|--------|--------|--------|
| = | 留T1 | 留T2 | 留T3 |
| ↑ | - | 晋级T1 | 晋级T2 |
| ↓ | 降级T2 | 降级T3 | - |
| max1 | 留T1 | 晋级T1 | - |
| min1 | 留T1 | - | - |
| min2 | 降级T2 | 留T2 | - |
| max2 | - | 留T2 | 晋级T2 |
| min3 | - | 降级T3 | 留T3 |
| max3 | - | - | 留在T3 |

---

## T3→T2 特殊检查

当从 T3 降级到 T2 时，检查 T1+T2 主位总数是否小于配置的主位总和：
- 如果 T1+T2 主位总数 >= (T1.limit + T2.limit)，拒绝进入 T2 缓冲位，保持原位

通用化：当任何梯队需要降级到缓冲位时，检查所有更高梯队的主位空位数之和是否为0。
