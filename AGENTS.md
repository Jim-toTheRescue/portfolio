# 开发指南

## 命令规则
- 任何有副作用的操作都需要用户确认之后再执行
- **git 操作只能由用户要求时执行，不能独立发起**

## 测试
- 用户自己开 dev server，不需要 build
- 直接刷新浏览器测试即可
- 浏览器打开 localhost:3000

## 经验教训
- 出现问题先检查代码，不要怀疑浏览器/缓存
- 问题在哪个文件就直接去那个文件找，不要用grep乱搜
- 不要擅自决定"只有什么情况才做什么"，只有用户明确要求才修改
- 改代码前先确认是否偏离用户需求
- 加日志要准确，直接在问题代码处加
- 不要猜测，有问题直接问用户
- 如果反复定位数据和代码都找不出问题一直死循环，可以请求用户提供日志协助定位问题
