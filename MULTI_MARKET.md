# 多市场股票支持 - 需求文档

## 1. 背景与目标

### 1.1 背景

当前 Portfolio 系统**仅支持美股**（通过 `qt.gtimg.cn/q=us{symbol}` API）。

需要扩展支持：
- **港股**（如 00700.HK）
- **A股**：上交所（SH）、深交所（SZ）

### 1.2 目标

在同一个组合中同时持有美股、港股、A股，进行统一的仓位管理。

---

## 2. 市场与代码

### 2.1 市场识别

用户输入股票代码后**自动识别市场**（根据代码格式）：

| 市场 | 代码后缀 | 格式特征 | 示例 | 存储symbol | API调用 |
|------|----------|----------|------|------------|---------|
| 美股 | .US | 纯字母 | AAPL | AAPL.US | usAAPL |
| 港股 | .HK | 5位数字 | 00700 | 00700.HK | hk00700 |
| 沪市 | .SH | 6位数字，以6开头 | 600519 | 600519.SH | sh600519 |
| 深市 | .SZ | 6位数字，以0或3开头 | 300750 | 300750.SZ | sz300750 |

### 2.2 腾讯 API

**支持混合市场一次性查询**：
```
https://qt.gtimg.cn/q=sh600519,sz300750,hk00700,usAAPL
```

返回结果按请求顺序排列。

## 3. 数据结构

### 3.1 持仓字段

存储 symbol 带后缀，通过后缀自动识别市场和货币：

```javascript
{
  "symbol": "600000.SH",
  "name": "浦发银行",
  "shares": 100,
  "price": 10.50,
  "value": 1050,          // 原始货币值
  "avgCost": 10.00,        // 成本价
  "tier": 3,
  "inBuffer": false
}
```

**工具函数**：
```javascript
function parseMarket(symbol) {
  const suffix = symbol.split('.').pop().toUpperCase();
  const marketMap = {
    'US': { market: 'US', currency: 'USD' },
    'HK': { market: 'HK', currency: 'HKD' },
    'SH': { market: 'SH', currency: 'CNY' },
    'SZ': { market: 'SZ', currency: 'CNY' }
  };
  return marketMap[suffix] || { market: 'UNKNOWN', currency: 'CNY' };
}
```

### 3.2 配置增加汇率

```javascript
{
  "portfolios": {
    "default": {
      "cash": 100000,       // 原始货币（CNY）
      "exchangeRates": {    // 汇率（以 USD 为基准）
        "USD": 1,
        "CNY": 7.10,
        "HKD": 7.75,
        "lastUpdate": "2024-01-01"
      },
      "autoFetchRates": true
    }
  }
}
```

## 4. 货币与汇率

### 4.1 多货币支持

| 市场 | 原始货币 |
|------|----------|
| 港股 | HKD |
| 美股 | USD |
| A股 | CNY |

### 4.2 汇率获取

**API**：
```
GET https://open.er-api.com/v6/latest/USD
```

**返回**（以 USD 为基准）：
```json
{
  "rates": {
    "USD": 1,
    "CNY": 7.10,
    "HKD": 7.75
  }
}
```

- 汇率缓存：每日更新一次

### 4.3 结算货币与显示货币

#### 结算货币（cashCurrency）
- **定义**：Portfolio 创建时选择的现金结算货币
- **用途**：用于所有仓位比较、占比计算、资金检查等**核心计算逻辑**
- **存储**：在 portfolio 的 `cashCurrency` 字段，默认为 CNY，可选 USD/HKD
- **特点**：创建后固定，仅在创建新 Portfolio 时选择

#### 显示货币（displayCurrency）
- **定义**：用户在头部下拉切换的显示货币
- **用途**：仅用于**界面展示**，不影响任何计算逻辑
- **特点**：运行参数，非持久化配置，切换后立即生效，不存储到 LocalStorage

#### 核心设计原则
1. **存储**：各市场股票以原始货币存储（港股 HKD，美股 USD，A股 CNY）
2. **计算**：所有仓位比较都转化为**结算货币**计算后比较
3. **展示**：由页面的"显示货币"选项决定，只用于显示不影响计算

#### 示例

假设：
- 结算货币：CNY
- 显示货币：USD
- 汇率：USD=1, CNY=7.10, HKD=7.75

持仓：
- 港股 00700：市值 HKD 10,000
- 美股 AAPL：市值 USD 1,000
- A股 600000：市值 CNY 10,000
- 现金：CNY 10,000

**计算（基于结算货币 CNY）**：
- 总资产 = 现金 CNY 10,000 + 持仓转换后 CNY
- 00700：HKD 10,000 × (7.10 ÷ 7.75) = CNY 9,159
- AAPL：USD 1,000 × 7.10 = CNY 7,100
- A股 600000：CNY 10,000
- 总资产：CNY 36,259
- 00700 占比：9,159 / 36,259 = 25.3%

**展示（基于显示货币 USD）**：
- 00700 显示：HKD 10,000
- AAPL 显示：USD 1,000
- 600000 显示：CNY 10,000 → USD 1,408
- 总资产显示：USD 5,107

**转换公式**：
- 原始货币 → 目标货币 = 原始货币 × (原始货币汇率 ÷ 目标货币汇率)

## 5. 功能

### 5.1 股票查询

- 用户输入股票代码
- 自动识别市场并追加后缀（如 `600519` → `600519.SH`）
- 调用腾讯 API 获取价格

### 5.2 价格刷新

- 批量获取不同市场的股票价格（混合市场一次性查询）
- 统一处理返回数据

### 5.3 持仓展示

- 市场标识（港/美/沪/深）
- 股票代码 + 市场（如 AAPL 美）
- 原始货币市值（如 `$1,234.56`）
- 转换后市值用于展示（如 `HKD 10,000 (≈ $1,290)`）
- 占比基于**结算货币**计算显示
- 涨跌幅（与原始货币一致）

### 5.4 建仓/调仓

- 输入股票代码自动识别市场
- 显示当前汇率，交易金额显示：原始金额 + 转换后金额（如 `HKD 10,000 ≈ $1,290`）
- 占比计算基于**结算货币 cashCurrency**
- 资金检查基于结算货币

### 5.5 历史记录

- 显示原始货币

## 6. UI 改动

### 6.1 头部显示

- 总资产（根据 displayCurrency 显示）
- 现金（根据 displayCurrency 显示）
- 货币切换下拉（CNY/USD/HKD）
- **货币图标随切换变化**：
  - CNY：¥
  - USD：$
  - HKD：hk$（因与美元符号相同，加 hk 前缀区分）
- 汇率设置入口

### 6.2 持仓卡片

- 市场标识
- 原始货币市值 + 转换后市值
- 涨跌幅

### 6.3 建仓弹窗

- 股票代码输入框（输入后自动识别市场）
- 显示市场类型
- 交易金额：原始金额 + 转换后金额 + 汇率（如 `HKD 10,000 ≈ $1,290 (汇率 7.75)`）
- 推荐股数按钮

### 6.4 汇率设置弹窗

- 自动获取汇率按钮
- 手动输入 USD/CNY、HKD/CNY 汇率
- 显示最后更新时间

## 8. 风险与限制

1. A股可能无法通过海外API获取（需要翻墙或国内服务器）
2. 汇率波动影响市值计算
3. 需要处理非交易时段的价格显示
