# Portfolio 投资组合管理系统需求文档

## 1. 项目概述

### 1.1 项目背景
这是一个基于Web的投资组合管理系统，用于管理股票投资仓位。该系统采用三层梯队（Tier）制度，帮助投资者将持仓按市值比例分配到不同梯队，实现仓位管理自动化。

### 1.2 核心功能
- 三层梯队持仓管理
- 智能调仓计算（多种按钮模式）
- 自动再平衡
- 实时价格刷新
- 数据持久化（LocalStorage）
- 导入/导出功能

---

## 2. 梯队系统定义

### 2.1 梯队配置

| 梯队 | 名称 | 目标比例 | 比例范围 | 主位上限 | 缓冲位上限 | 下限 | 上限 |
|------|------|-----------|----------|----------|------------|------|------|
| T1 | 第一梯队 | 30% | ≥25% | 1 | 0 | 25% | 35% |
| T2 | 第二梯队 | 20% | ≥15% 且 <25% | 2 | 1 | 15% | 25% |
| T3 | 第三梯队 | 10% | ≥5% 且 <15% | 3 | 3 | 5% | 15% |

### 2.2 边界规则
- **25%** 属于 T1（第一梯队）
- **15%** 属于 T2（第二梯队）
- **5%** 属于 T3（第三梯队）

### 2.3 梯队判定逻辑
```
function getTargetTier(percent):
  if percent >= 25 return T1
  if percent >= 15 return T2
  return T3
```

## 3. 核心功能模块

### 3.1 持仓管理

#### 3.1.1 显示持仓
- 按梯队分组展示（三个卡片）
- 每个持仓显示：股票代码、名称、股数、市值、单价、涨跌幅
- 显示 Drift（偏离度）：当前占比与目标占比的差值
- 超出范围提示（颜色标记：绿色=正常，黄色=警告，红色=严重）
- 建议操作提示（升/降梯队建议）

#### 3.1.2 主位与缓冲位
- 每个梯队有主位（main slots）和缓冲位（buffer slots）
- 当主位满时，新进入该梯队的持仓进入缓冲位
- 缓冲位可升级到主位

#### 3.1.3 空位显示
- 未填满的主位显示为"空位"
- 缓冲位显示为"缓冲"

### 3.2 建仓功能

#### 3.2.1 建仓入口
- 点击 "+ 建仓" 按钮打开弹窗

#### 3.2.2 股票查询
- 输入股票代码后自动获取：
  - 股票名称
  - 当前价格
- 数据源：腾讯财经API（`qt.gtimg.cn`）

#### 3.2.3 股数输入
- 手动输入
- 推荐按钮：
  - **↑**：推荐到最低梯队目标比例
  - **max**：推荐到最高允许建仓梯队（topTierAllowBuy）的上限比例

#### 3.2.4 最高允许建仓梯队
- 在创建 Portfolio 时设置 "最高允许建仓梯队"（topTierAllowBuy）
- 该设置限制新股票最多只能建仓到第 N 梯队
- 建仓面板显示当前最高建仓比例提示
- 验证逻辑：
  - 单笔不超过 topTierAllowBuy 梯队的上限
  - topTierAllowBuy 梯队主位/缓冲位已满时不能建仓

#### 3.2.5 交易金额显示
- 实时显示：交易金额（美元）+ 占总资金比例
- 超出资金提示

#### 3.2.6 建仓验证
- 不能超过 topTierAllowBuy 梯队的上限
- 不能超出可用资金
- topTierAllowBuy 梯队主位/缓冲位已满时不能建仓

#### 3.2.7 梯队分配
- 新建仓位根据市值比例自动判定梯队
- 自动加入对应梯队的主位或缓冲位

### 3.3 调仓功能

#### 3.3.1 调仓入口
- 每个持仓显示"加仓"/"减仓"按钮
- 点击打开调仓弹窗

#### 3.3.2 调仓弹窗信息
- 显示当前持仓信息（股数、市值、占比）
- 操作股数输入框
- 交易金额显示
- 交易后仓位预览（股数 + 占比）
- 警告信息（如有）

#### 3.3.3 调仓按钮说明

##### 3.3.3.1 =（等于）按钮
调整到当前梯队目标比例。

| 持仓梯队 | 目标 | 操作 | 结果 |
|----------|------|------|------|
| T1 | 30% | 自动判断加/减 | 留在T1 |
| T2 | 20% | 自动判断加/减 | 留在T2 |
| T3 | 10% | 自动判断加/减 | 留在T3 |

##### 3.3.3.2 ↑（向上箭头）按钮
调整到上一梯队（更高梯队）的目标比例。

| 持仓梯队 | 目标 | 操作 | 结果 |
|----------|------|------|------|
| T2 | 30%（T1目标） | 加仓 | 晋级T1 |
| T3 | 20%（T2目标） | 加仓 | 晋级T2 |
| T1 | - | 不可用 | - |

##### 3.3.3.3 ↓（向下箭头）按钮
调整到下一梯队（更低梯队）的目标比例。

| 持仓梯队 | 目标 | 操作 | 结果 |
|----------|------|------|------|
| T1 | 20%（T2目标） | 减仓 | 降级T2 |
| T2 | 10%（T3目标） | 减仓 | 降级T3 |
| T3 | - | 不可用 | - |

##### 3.3.3.4 max1 / min1 / min2 按钮（T1持仓可用）

| 按钮 | 目标 | 操作 | 结果 |
|------|------|------|------|
| max1 | <35%（T1上限） | 加仓 | 留在T1 |
| min1 | 25%（T1下限） | 减仓 | 留在T1 |
| min2 | 15%（T2下限） | 减仓 | 降级T2 |

##### 3.3.3.5 max2 / max1 / min2 / min3 按钮（T2持仓可用）

| 按钮 | 目标 | 操作 | 结果 |
|------|------|------|------|
| max2 | <25%（T2上限） | 加仓 | 留在T2 |
| max1 | 35%（T1上限） | 加仓 | 晋级T1 |
| min2 | 15%（T2下限） | 减仓 | 留在T2 |
| min3 | 5%（T3下限） | 减仓 | 降级T3 |

##### 3.3.3.6 max3 / max2 / min3 按钮（T3持仓可用）

| 按钮 | 目标 | 操作 | 结果 |
|------|------|------|------|
| max3 | <15%（T3上限） | 加仓 | 留在T3 |
| max2 | <25%（T2上限） | 加仓 | 晋级T2 |
| min3 | 5%（T3下限） | 减仓 | 留在T3 |

#### 3.3.4 按钮汇总表

| 按钮 | T1持仓 | T2持仓 | T3持仓 |
|------|--------|--------|--------|
| = | 留T1 | 留T2 | 留T3 |
| ↑ | - | 晋级T1 | 晋级T2 |
| ↓ | 降级T2 | 降级T3 | - |
| max1 | 留T1 | 晋级T1 | - |
| min1 | 留T1 | - | - |
| min2 | 降级T2 | 留T2 | - |
| max2 | - | 留T2 | 晋级T2 |
| min3 | - | 降级T3 | 留T3 |
| max3 | - | - | 留在T3 |

#### 3.3.5 调仓计算逻辑（重要约束）

**每一个按钮推荐的股票数，都必须最接近梯队规则限制值。**

即：如果推荐 `adjShares`，那么这个值交易出来的新仓位需要比 `adjShares-1` 和 `adjShares+1` 更接近梯队限制值。而 `adjShares-1` 和 `adjShares+1` 要么比 `adjShares` 离梯队限制值更远，要么直接溢出当前梯队了。

### 3.4 清仓功能

#### 3.4.1 清仓入口
- T3持仓显示"清仓"按钮

#### 3.4.2 清仓操作
- 弹出确认对话框
- 确认后：
  - 卖出全部持仓
  - 现金增加持仓市值
  - 持仓从列表移除

### 3.5 自动再平衡

#### 3.5.1 触发时机
- 每次加仓/减仓/清仓操作后自动执行

#### 3.5.2 再平衡逻辑
1. 根据当前市值计算每个持仓的占比
2. 根据占比重新判定目标梯队
3. 按市值从高到低排序，依次分配主位和缓冲位
4. 优先级：高市值持仓优先占据主位

#### 3.5.3 分配规则
1. 首先尝试进入目标梯队主位（如果主位有空位）
2. 如果主位已满，尝试进入目标梯队缓冲位（如果缓冲位有空位，且满足特殊检查）
3. 如果主位和缓冲位都满了，保持原位

#### 3.5.4 梯队槽位限制
- T1: 主位 1, 缓冲位 0
- T2: 主位 2, 缓冲位 1
- T3: 主位 3, 缓冲位 3

#### 3.5.5 T3→T2 特殊检查
- 当从 T3 降级到 T2 时，检查 T1+T2 主位总数是否 < 3
- 如果 T1+T2 主位总数 >= 3，拒绝进入 T2 缓冲位，保持原位

## 4. 辅助功能

### 4.1 价格刷新

#### 4.1.1 入口
- 头部"刷新价格"按钮

#### 4.1.2 功能
- 刷新所有持仓的当前价格
- 显示最后刷新时间

### 4.2 现金管理

#### 4.2.1 显示
- 头部摘要区域显示现金金额及占比

#### 4.2.2 修改
- 点击现金金额弹出修改弹窗
- 输入新现金值并确认

### 4.3 导入/导出

#### 4.3.1 导出
- 点击"导出"按钮
- 下载JSON文件（包含所有持仓和现金数据）

#### 4.3.2 导入
- 点击"导入"按钮
- 选择JSON文件
- 数据覆盖当前数据

### 4.4 Mock价格

#### 4.4.1 入口
- 头部"Mock价格"按钮

#### 4.4.2 功能
- 输入JSON格式的价格数据
- 批量设置指定股票的价格
- 用于测试场景

### 4.5 历史记录

#### 4.5.1 显示
- 点击"历史"按钮展开历史面板
- 显示最近20条操作记录

#### 4.5.2 记录内容
- 操作类型（加仓/减仓/清仓/建仓）
- 股票代码
- 操作股数
- 成交价格
- 梯队变化（晋级/降级）
- 操作时间

#### 4.5.3 清除历史
- 点击"清历史"按钮清除所有历史记录

## 5. 数据存储

### 5.1 LocalStorage

#### 5.1.1 持仓数据
- Key: `portfolio`
- 格式：
```json
{
  "positions": [
    {
      "symbol": "AAPL",
      "name": "Apple Inc.",
      "shares": 100,
      "price": 150.00,
      "value": 15000,
      "tier": 1,
      "inBuffer": false,
      "avgCost": 145.00
    }
  ],
  "cash": 50000,
  "priceTime": "2024-01-01 10:00:00"
}
```

#### 5.1.2 历史数据
- Key: `portfolio-history`
- 格式：数组，每条记录包含操作类型、股票代码、操作股数、价格、时间等

## 7. 边界情况处理

### 7.1 资金不足
- 加仓时如果所需资金超过现金，提示"资金不足"

### 7.2 持仓上限
- 单一持仓不能超过35%，提示"超出上限35%"

### 7.3 股票未找到
- 输入的股票代码在API中未找到，提示"未找到"

### 7.4 调仓后股数为0
- 减仓导致股数为0时，自动清仓该持仓
